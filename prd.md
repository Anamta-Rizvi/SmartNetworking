# 📘 Campus OS — Product Requirements Document

## 🚀 Overview

Campus OS is an intelligent campus platform that turns campus opportunities into **structured, measurable outcomes**. It combines:

- 📅 **Event Discovery** — aggregated, goal-ranked, color-coded by relevance
- 🤖 **AI Copilot** — structured goal setup → quantified milestones → event + connection recommendations
- 📊 **Progress Tab** — charts for RSVPs, attendance, referrals; referral log with company; milestone tracking
- 🗺️ **Live Campus Map** — events color-coded by goal relevance; friend RSVP presence
- 🤝 **Connections + Chat** — goal-driven suggestions, open search, direct messaging, AI referral prompts
- 🎓 **Class Schedule Awareness** — daily plan and event suggestions respect your weekly class schedule
- 🏢 **Company Preferences** — preferred companies + job roles; AI surfaces relevant companies and connections

---

## 🎯 Vision

> Build an operating system for student life that converts campus opportunities into real outcomes — jobs and skills, **and** friendships, community, and repeat habits worth showing up for.

---

## ❗ Problem

Students:
- Attend events randomly without connecting them to outcomes
- Have no structured way to track progress toward career or social goals
- Struggle to identify *which* events, people, or resources will actually move them forward
- Waste time at low-relevance events without knowing it
- Can't see how close they are to hitting targets (referrals, connections, event attendance)
- Miss networking opportunities with people they already connected with at events

---

## 💡 Solution

Campus OS uses a **goal-first architecture**:
1. Copilot asks structured questions → understands the student's goal, timeline, and field
2. AI generates **quantified targets** (e.g. 15 referrals, 10 career events, 20 connections in 3 months)
3. Every event is **scored and weighted** against those targets
4. Map shows events **color-coded by goal relevance**
5. Progress tab tracks real progress with charts, referral logs, and milestone bars
6. Connection suggestions are **entirely goal-driven** (alumni, industry professionals, peers in same track)
7. Chat with connections; AI proactively prompts you to ask for referrals from recent connections
8. Daily plan respects your **class schedule** — no suggestions during class time

## 🤖 Copilot — Structured Goal Setup

**NOT free-form chat.** Goal Setup is a **fixed questionnaire flow** — one question at a time, with selectable options + optional free text for each answer.

### Questions (in order)

**Q1 — Primary Goal**
> "What's your main goal right now?"
- 🎯 Land a job / internship
- 🌱 Explore career options
- 🤝 Grow my professional network
- 🎉 Make friends and find community
- ⚡ All of the above

**Q2 — Timeline** *(if career goal selected)*
> "When do you need to achieve this by?"
- In the next month
- In the next 3 months
- By end of semester
- By graduation
- No deadline

**Q3 — Field / Industry** *(if career goal)*
> "What field are you targeting?"
- Software Engineering / Tech
- Finance / Investment Banking
- Consulting
- Product Management
- Marketing / Growth
- Healthcare / Life Sciences
- Other: [text input]

**Q4 — What kind of help do you need most?** *(if career goal)*
> "What's your biggest gap right now?"
- Getting referrals from professionals
- Building interviewing skills
- Finding mentors / advisors
- Meeting peers in the same field
- Understanding the industry better
- Multiple of the above

**Q5 — Social goal detail** *(if social or both)*
> "What kind of social life are you building?"
- Find people with shared hobbies
- Build a friend group to hang out with
- Meet people across different majors/backgrounds
- Join clubs or recurring communities
- Other: [text input]

**Q6 — Confirmation + Dashboard Creation**
> Copilot summarizes: "Based on your answers, here's your 3-month plan: [targets]. I'll track your progress and suggest the best events and people to help you hit these goals. Ready to start?"
- ✅ Looks good — Create my dashboard
- ✏️ Let me adjust something

### AI-Generated Targets (examples by goal type)

| Goal | Targets Generated |
|---|---|
| Land a job in 3 months | 15 referrals, 10 career events, 20 professional connections, 5 alumni conversations |
| Grow network (no deadline) | 5 events/month, 10 new connections/month |
| Make friends + find community | 3 social events/week, 8 new connections, 2 clubs joined |
| Explore career options | 2 info sessions/week, 5 professionals met, 3 fields explored |

Targets are stored as `GoalMilestone` records with `target_count`, `current_count`, `unit` (events / connections / referrals).

## 📅 Daily Plan — Schedule-Aware Event Suggestions

The AI Daily Plan is generated by the Copilot's `daily_planner` mode and is **aware of your weekly class schedule**.

### How it works
1. User inputs their weekly class schedule (day, start time, end time, class name) in Preferences
2. Daily planner fetches today's events from the database
3. AI cross-references event times against class schedule:
   - **Free slot**: event shown normally with "Accept / Reject" buttons
   - **Class conflict**: event flagged with "Conflicts with [ClassName] — attend anyway?"
4. User taps **Accept** → event added to their daily plan + goal dashboard
5. User taps **Reject** → event dismissed from today's suggestions

### UI
- AI generates a prioritized list of today's events with time slots
- Each event card shows:
  - Title, time, location
  - Goal relevance score
  - **Free / Conflict badge**
  - **Accept** (green) / **Reject** (red) buttons
  - If conflict: "Conflicts with [ClassName] — still want to go?" with Accept/Reject
- Accepted events appear in the "Your Plan" section at the top
- Plan updates in real time as user accepts/rejects

## 🤝 Networking — Event Preparation Mode

The Copilot `networking` mode has been redesigned around **preparing for events you already RSVPed to**.

### Flow
1. Copilot shows a list of your upcoming RSVPed events
2. User selects one → "Prepare for [Event Name]"
3. AI generates a preparation brief including:
   - **Who to meet (professionals)**: name, company, role, why relevant to your goal
   - **Who to meet (students)**: name, major, shared interests, what to talk about
   - **Suggested conversation topics** based on their background + your goal
   - **Goal relevance**: how this event can move your metrics
4. No elevator pitch, icebreaker, or follow-up templates — focus is on event-specific prep

### Preparation Brief format
```
Event: [Name] — [Date & Time]

Professionals to meet:
- Jane Smith (Google, Software Engineer) — Recruiting for backend roles, attended 3 similar events
- Mark Johnson (Goldman Sachs, Analyst) — Works in fintech, alumni of your school

Students to connect with:
- Alex Chen (CS, Class of 2026) — Interested in ML, going to same career fair next week
- Priya Patel (Finance, Class of 2025) — Active in FinTech club, 2 shared RSVPs

Conversation angles:
- Ask Jane about Google's internship timeline
- Mention your shared interest in distributed systems with Alex
```

## 🏆 Referral Tracking

### Logging a referral
From the **Progress tab** or **Goal Dashboard**, user can:
- Tap "Log Referral Received"
- Enter: Company name, contact name (optional), notes (optional)
- Referral is timestamped and saved
- Referral milestone counter increments automatically

### Asking about referrals
After each attended event (within 48 hours), Copilot prompts:
> "Did you receive any referrals at [Event Name]?"
- Yes → log referral flow
- No → dismiss

## 🏢 Company Preferences

In the **Preferences / Profile** screen, users can set:

### Preferred Companies
- Add companies they're targeting (e.g., Google, McKinsey, JPMorgan)
- For each company, select the role/function they're interested in
- This is saved as `CompanyPreference` records

### AI Behavior
- Copilot daily plan and networking mode **prioritize events** hosted by or connected to preferred companies
- Connection suggestions **highlight alumni or employees** from preferred companies first
- AI also suggests **relevant similar companies** the user hasn't listed (e.g., if you have Google, also suggest Meta, Amazon)
- In the Connections screen, preferred-company contacts get a "Target Company" badge

## 💬 Chat with Connections

### Direct Messaging
- Users can send direct messages to accepted connections
- Messages are stored in `DirectMessage` table
- Chat screen shows: conversation list (most recent first), unread badge
- Conversation screen: message bubbles (sent/received), timestamp, "Send" button

### AI Referral Nudge
The AI proactively surfaces a **referral suggestion banner** at the top of the Chat screen:

> "You connected with 5 people at [Event Name] yesterday. Want to ask them for a referral?"

- Banner shows the list of people connected with at that event
- User can **select** which ones to message (multi-select checkboxes)
- Tap "Send with AI" → AI drafts a personalized referral request message for each selected person and sends it automatically
- Message is personalized based on: shared event, their company, user's goal/field

### AI-Drafted Message Example
```
Hey [Name]! It was great connecting with you at [Event] yesterday.
I'm really interested in opportunities at [Company] in [Role].
Would you be open to referring me or sharing any advice?
Thanks so much!
```

### Message Modes
- **Manual**: user types their own message
- **AI-assisted**: AI drafts, user reviews and sends

## 📊 Goal Dashboard

The Goal Dashboard is now a sub-section within the Progress tab (not a separate bottom tab).

### Metrics tracked

**Career track**
- Referrals received (`referral_count`) — with company log
- Career events attended (`career_events_attended`)
- Professional connections made (`professional_connections`)
- Alumni conversations had (`alumni_conversations`)

**Social track**
- Social events attended
- New friends made
- Clubs / communities joined
- Recurring event attendance (habit streak)

### Event Weightage System

Each event has a `contribution_score` (0–1) per goal type:

| Event type | Referral weight | Career event weight | Connection weight |
|---|---|---|---|
| Career fair | 0.9 | 1.0 | 0.6 |
| Alumni networking | 0.8 | 0.7 | 0.8 |
| Industry panel | 0.5 | 0.8 | 0.4 |
| Workshop / skill | 0.2 | 0.7 | 0.2 |
| Social mixer | 0.1 | 0.1 | 0.7 |
| Hobby / club event | 0.0 | 0.0 | 0.5 (social) |

## 🗺️ Live Campus Map — Goal-Relevance Color Coding

Event pins on the map are **color-coded by their relevance to the user's active goal**:

| Relevance score | Color | Meaning |
|---|---|---|
| 80–100% | 🟣 Purple (primary) | Directly advances your goal — high priority |
| 50–79% | 🟡 Amber | Partially relevant — worth considering |
| 20–49% | 🔵 Blue | Low relevance to your goal |
| < 20% | ⚫ Gray | Not relevant to your current goal |

## 🤝 Connections — Goal-Based Suggestions vs Open Search

### Connection Suggestions (goal-driven)
- **Career goal**: alumni in target field → industry professionals → peers with same major
- **Social goal**: users who RSVPed to same hobby/social events → users with overlapping interest tags
- **Company preference boost**: connections from preferred companies ranked higher
- Suggested user cards show **why** they're suggested + "Target Company" badge if from preferred company

### Name / Email Search (open, goal-blind)
- Searches by name, email, or major across all users at the same university
- No goal filtering — if the user exists, they appear

### Chat Button
- Each connection card has a "Message" button → opens ConversationScreen

# 🧠 AI Components

- **Goal questionnaire engine**: structured Q&A → milestone generation → target numbers
- **Event relevance scorer**: per-user, per-event score based on goal type + field + milestone gaps
- **Dashboard contribution calculator**: event weight × milestone target → % contribution
- **Connection suggestion ranker**: goal-context scoring (alumni match, field match, event overlap, company preference)
- **Daily planner**: today's events cross-referenced with class schedule → free/conflict badges → accept/reject
- **Event prep brief generator**: for RSVPed events — who to meet, conversation angles, goal fit
- **Referral nudge**: post-event prompt asking if referral received
- **AI referral message drafter**: personalized outreach to recent connections asking for referral
- **Company suggestion engine**: given preferred companies, suggests similar relevant companies + connections

---

# 🏗️ Architecture

- FastAPI + SQLAlchemy + SQLite (dev) → PostgreSQL (prod)
- React Native + Expo 54
- Zustand (client state) + React Query (server state)
- Azure OpenAI (GPT-4.1, Whisper)

**Key models:**
- `User`, `Goal`, `GoalEvent` (event-to-goal linkage with contribution score)
- `Event` (with `goal_relevance_score` computed per-user), `EventTag`
- `Connection` (bidirectional, pending/accepted/declined)
- `ClassSchedule` (weekly class slots per user — day, start_time, end_time, class_name)
- `CompanyPreference` (user's target companies + job roles)
- `Referral` (logged referrals with company, contact, date)
- `DirectMessage` (sender, receiver, content, is_ai_generated, read_at)
- `RSVP`, `UserInterest`, `UserLocation`, `NotificationPreference`
